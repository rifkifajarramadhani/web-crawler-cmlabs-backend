Google Search Console (GSC) is a free tool from Google that helps web owners to monitor performance, traffic, and fix problems on their websites. This tool comes with a number of data such as the number of impressions, total clicks, average CTR, average position, website performance on mobile devices, and more. GSC, previously known as Google Webmaster Tools, continues to develop its features. The feature has many benefits for web owners or bloggers who need SEO tools to monitor and optimize their website performance. Google Search Console features As previously mentioned, the GSC is equipped with quite complete features. Google Search Console features offer the Overview, Performance, URL Inspection, Index, Page Experience, and Security. Find the explanation of each of the Google Search Console features below: Overview The overview feature in GSC contains four main reports consisting of Performance Report: shows a summary of the traffic you've had in the last few months. To view the entire report you can press the Full Reports option. Index Coverage (Coverage): contains a summary of all your web pages indexed by Google. Click Full Reports to view the complete report. Page Experience Report: shows a summary of performance on desktop and mobile devices, the assessment of core web vitals, and page experience on your website. Page Enhancement Report: contains a summary of page enhancements such as search optimization or AMP (Accelerated Mobile Pages) implementation. You'll also see information on whether page enhancements went well or if there are still errors. Figure 2: An overview report containing a summary of four reports consisting of a summary of performance, index coverage, page experience, and page enhancements. You can view the full report by pressing the Full Reports option. Performance The performance tool shows the amount of traffic you managed to get from Google search results. This feature allows you to view traffic by query, page, country, device type, and more. When you open the performance report, you'll see the following four SEO metrics: Total Clicks: shows how many people clicked on your web page in Google search results. Total Impressions: provides information on the number of times your web page appears in the Google SERP. Average CTR: displays the percentage of the impressions your web page has that resulted in a click. Average Position: contains information about the average position of your web page in Google search results. Figure 3: Website performance report view showing total clicks, total impressions, average CTR, and average position of a web page on Google search. This Google Search Console feature also allows you to view traffic based on certain categories. URL Inspection The URL inspection tool provides information about specific pages on your website that Google has indexed successfully. The information includes AMP issues, structured data issues, and other indexing issues. The functions of the URL inspection feature found in Google Search Console are as follow: Displays URL index status: provides information on whether your web page was successfully indexed by Google or not, along with the reason. Checking the active URL: checks if the pages on your website can be indexed. Request URL indexing: allows you to send Google a recrawl request. Show the rendered version of the page: provides a screenshot of how your web page will look in Google's eyes. View loaded web resources: lets you view resource lists, page code, javascript output, and other info. Troubleshoot missing pages: URL inspection can help you find the cause of the page not being indexed. Shows the last crawl time: provides information about the last time Google crawled your web page.   Index The index feature in GSC consists of three main reports including index coverage, sitemaps, and a list of removed URLs. Check out the explanation of the reports function: Index Coverage (Coverage) The indexing coverage report shows which pages of your site Google succeeded and failed to index. In addition, this feature also comes with information regarding indexing issues found on your site. There are 4 assessment statuses that will be displayed on the coverage index report, namely: An Error Occurred: displays the number of URLs of the page that Google could not index for some reason. You can prioritize this page for optimization. Valid with warnings: shows the total URL of the page that was indexed successfully by Google, but there are some errors in it. Valid: provides information on the number of page URLs that were successfully indexed in Google search results. Excluded: displays the total of pages that Google intentionally didn't index. This can happen when you intentionally add a noindex command or decide a page is a duplicate of another canonical page, then Google won't index the page.   Each index rating status will have a specific reason that caused the indexing problem to occur as well as recommendations on how to fix it. In addition, Google also allows GSC users to share and export this index coverage report data. Figure 4: An index coverage report display containing information on the number of error URLs, valid URLs with a number of warnings, total valid URLs, and total excluded URLs. Sitemaps The sitemap feature in Google Search Console helps you to view the sitemaps that Google has found, add new sitemaps to your site, view the history of sitemap additions, and identify errors in your site's sitemap. Reports displayed for each sitemap added in GSC include: Sitemap URL: contains the URL where the sitemap is added, usually associated with the property root. Sitemap type: provides sitemap format information. Possible values ​​are sitemap (XML or text), sitemap index (sitemap of multiple sitemaps), RSS, Atom, and unknown (sitemap type unknown or unprocessed). Delivery time: displays the date the sitemap was last submitted to Google. Last Crawl Time: indicates the last time the sitemap was processed by Google. Status: the crawling status of the sitemap by Google. Possible values ​​are successful, contain errors, and cannot retrieve sitemap data. Found URL: displays the total number of URLs contained in the sitemap. URL Removal (Removals) GSC allows you to temporarily block a page's URL from Google search results, view a history of user-submitted URL removal requests, and view URLs on your website that are reported as adult content with Google's Safe Search filters. This feature can only remove URLs in Google search results and cannot be applied to other search engines. You can take advantage of this tool to troubleshoot crawl issues in your GSC account, get rid of hacked URLs, fix duplicate content issues, and more. Page Experience Page experience is one of the ranking factors in Google's SERP. Therefore, you should carry out regular monitoring and improve the quality of the user experience on the web pages that you have. That way, users will have a pleasant experience on your site and your site's ranking will increase. The page experience feature in Google Search Console groups three reports consisting of page experience, core web vitals, and mobile usability. Take a look at the presentations of the three reports below: Page Experience The page experience report shows a summary of the user experiences who visited your website. Google performs an assessment of page experience metrics on each URL on your site as one of Google ranking factors. The data displayed includes the percentage of good URLs, total good URL impressions, and the daily value of good URLs percentage. Figure 5: Display of a page experience report that summarizes the number of URLs with a good status, total impressions, total URL errors on core web vitals, and HTTPS implementation status to maintain site security. Core Web Vitals The core web vitals report deals with the assessment of a website's performance on both desktop and mobile rankings. With this report, you can track the performance of URLs categorized by status (good, need of improvement, or poor), metric types (FID, LCP, and CLS), and URL groups (groups of similar pages). Only indexed URLs are shown in this report. The three main metrics in core web vitals are First Input Delay (FID), Largest Contentful Paint (LCP), and Cumulative Layout Shift (CLS). The performance of a website can be said to be good if the site manages to reach the minimum score on these three metrics. Mobile Usability The mobile usability report provides information regarding issues your website is experiencing accessed on a mobile device. If it is found that your site does not support page responsiveness on mobile devices, then you should fix it immediately to improve the user experience. Manual Action & Security The manual actions and security features provide information regarding violations that occurred on your website so that the page gets a warning from Google. A website will get a penalty from Google if it loads a page that doesn't comply with Google's guidelines. How to Use Google Search Console Besides understanding the basic concepts of GSC, you also need to do hands-on practice so you can monitor the performance and optimize your website. In the following passage, we will show you how to use Google Search Console. Adding Sites to GSC The first thing is that you have to add the site to the GSC. To complete the action, you need to log in using email to the GSC page. After successfully logging in, click the “Add property” option, then you will be required to enter the domain address in the field provided. Figure 6: An image showing the steps for adding a site to Google Search Console, by pressing Add Property then a form will appear to add a domain and URL prefix. Next, you need to verify the domain. Here are 4 ways to verify your site on Google Search Console: HTML File: You can choose the HTML File verification method via GSC, then download the file by clicking on the file in the download the file section. The next step is to upload the HTML file on your domain then return to the GSC page and click Verify. TXT Record: You need to add a custom TXT Record in your domain settings to perform website verification. HTML Tag: Select the HTML tag verification method in GSC, then copy the HTML code snippet of the tag into the tag of your site's page code. Google Analytics Tracking Code: You can also do verification with Google Analytics. Open Google Analytics and select the Property Settings menu. Then click Tracking Info and copy and paste your tracking code into the tag of your website.   Setting the Target Country You can set which countries you target to instruct so that your site appears on search results in the country you choose. To set the target country, you need to select the Legacy tools and reports menu, then click the International Targeting sub-menu and open the Country tab. Next, determine which country is your target market. Setting a target country aims to help Google better understand your primary target audience. However, your site may appear in search results of other countries if it is relevant to the user's search query. Figure 7: Display of the country settings menu to be targeted. When you click on the menu, you can select the country that is your target market in Google Search Console. Connecting GSC and Google Analytics To connect GSC and Google Analytics is by press the Admin menu on the Google Analytics dashboard, then selecting Property Settings, and then press the "Adjust Search Console" button. Google Analytics will direct you to the Google Search Console page along with the entire list of websites you have monitored. Select a website that you want to connect with Google Analytics, then press the “Save” button and Google Analytics will be linked to your GSC account. Adding Sitemap The greater the size of the website, the more pages and content it has. Great websites usually have more than 100 web pages. If your site is included in a great website, you need to add a sitemap to Google to facilitate the crawling and indexing process. If you don't have a sitemap yet, then you can use the Sitemap Generator tool from cmlabs. Here is how to add a sitemap in Google Search Console: Select the Index menu in GSC, then you will see several sub-menus such as Coverage, Sitemaps, and Removals. To add a sitemap, click the Sitemaps sub-menu. Next, copy your sitemap URL and click the “Submit” button to add the sitemap to Google. GSC will display a message on whether or not the addition of the sitemap was successful. Figure 8: Image of the stages of adding a sitemap in Google Search Console. Open the Sitemaps menu, then add the URL where the sitemap is stored, and press the Submit button. You will see a history of the sitemaps that have been added along with their status. Requesting an Indexing With Google Search Console, you can also request indexing. Click the URL inspection menu then enter the URL address of the web page you want to index. After that, GSC will display detailed information on whether the URL you entered has been indexed or not. If the URL you entered is not indexed, you can submit a re-indexing request by pressing the “Request Indexing” option. Figure 9: Display of the URL inspection menu in Google Search Console. When you click on the menu, you will be required to enter the URL you want to search. Then you will see a report on whether the URL was indexed successfully or not by Google. Adding a User GSC also supports team collaboration. You can add multiple users to one website monitoring project. Here's how to use Google Search Console to add multiple users: Open GSC and select the Settings menu then GSC will direct you to the general settings page. Figure 10: Display of the Settings menu on Google Search Console. Clicking the button will display the general settings page. On that page, select the Users and Permissions option, then click the “Add User” option. GSC will ask you to complete the user data to be added, including email and the type of permission granted. Figure 11: Display of user list contained in Google Search Console. You can add other users by pressing the “Add User” button in the top right corner. Enter the user’s email address and set the Permission type for that user. The permission options that can be given are Owner (property owner), Full (full access rights for all GSC features), and Restricted (access rights to view data, but cannot add other users). Figure 12: Display of a form to add a user's email address and the type of permissions to be granted to that user. After filling out the form you can click the “Add” button to add a user or press “Cancel” to cancel. Successfully added users have access to monitor website performance in Google Search Console according to the permissions given. All in all, that's an in-depth guideline of Google Search Console, starting from the definition, description of each feature, and how to use it. You can use this tool to add a sitemap, monitor site performance, and detect crawling and indexing issues. Make sure to always check the site regularly through the GSC.
